백준 14719번 **빗물** 문제는 코딩 테스트에서 정말 자주 나오는 'Trapping Rain Water' 유형의 클래식 문제입니다.

이 문제는 2차원 공간 전체를 훑는 것보다, **"특정 위치(세로줄)를 기준으로 이 칸에 물이 얼마나 고일까?"**를 1차원적으로 생각하면 아주 쉽게 풀립니다.

### 💡 핵심 아이디어: "내 양쪽으로 가장 높은 벽이 누구지?"

어떤 특정 위치에 물이 고이려면, 반드시 **왼쪽과 오른쪽에 현재 위치보다 높은 벽**이 있어야 합니다. 물은 양쪽 벽 중 '더 낮은 벽'의 높이까지만 찰 수 있습니다.

1. **양 끝단 제외:** 맨 왼쪽(인덱스 0)과 맨 오른쪽(인덱스 W-1)은 벽이 없으므로 물이 밖으로 흘러넘칩니다. 여긴 검사할 필요가 없습니다.
2. **현재 위치 기준 최대 높이 탐색:** 인덱스 1부터 W-2까지 각 위치(`i`)마다 다음을 구합니다.
* `i`의 왼쪽 부분에서 가장 높은 블록의 높이 (`left_max`)
* `i`의 오른쪽 부분에서 가장 높은 블록의 높이 (`right_max`)


3. **고이는 물의 양 계산:**
* 물이 차오를 수 있는 한계선은 왼쪽 최고 높이와 오른쪽 최고 높이 중 **더 낮은 값**입니다. `boundary = min(left_max, right_max)`
* 만약 이 한계선(`boundary`)이 현재 내 위치의 블록 높이보다 높다면, 그 차이만큼 빗물이 고이게 됩니다.

### 1. Java 풀이

Java에서는 반복문을 통해 기준점의 왼쪽과 오른쪽의 최댓값을 각각 구해주면 됩니다. 전체 너비 $W$가 최대 500이므로, $O(W^2)$의 시간 복잡도로 돌아도 연산 횟수는 약 25만 번으로 시간 제한(1초) 내에 아주 여유롭게 통과합니다.

```java
import java.io.*;
import java.util.*;

public class Main {
    public static void main(String[] args) throws IOException {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        StringTokenizer st = new StringTokenizer(br.readLine());

        int H = Integer.parseInt(st.nextToken()); // 세로 길이 (사실상 이 풀이에서는 안 씀)
        int W = Integer.parseInt(st.nextToken()); // 가로 길이

        int[] blocks = new int[W];
        st = new StringTokenizer(br.readLine());
        for (int i = 0; i < W; i++) {
            blocks[i] = Integer.parseInt(st.nextToken());
        }

        int totalWater = 0;

        // 양 끝(0, W-1)은 물이 고일 수 없으므로 1부터 W-2까지만 검사
        for (int i = 1; i < W - 1; i++) {
            int leftMax = 0;
            int rightMax = 0;

            // 1. 현재 위치(i) 기준 왼쪽에서 가장 높은 벽 찾기
            for (int j = 0; j < i; j++) {
                leftMax = Math.max(leftMax, blocks[j]);
            }

            // 2. 현재 위치(i) 기준 오른쪽에서 가장 높은 벽 찾기
            for (int j = i + 1; j < W; j++) {
                rightMax = Math.max(rightMax, blocks[j]);
            }

            // 3. 물이 찰 수 있는 한계선은 양쪽 최고 벽 중 낮은 쪽
            int boundary = Math.min(leftMax, rightMax);

            // 4. 한계선이 현재 블록보다 높을 때만 물이 고임
            if (boundary > blocks[i]) {
                totalWater += (boundary - blocks[i]);
            }
        }

        System.out.println(totalWater);
    }
}

```

### 2. Python 풀이

파이썬에서는 리스트 슬라이싱과 내장 `max()` 함수를 활용하면 정말 직관적이고 짧게 구현할 수 있습니다.

```python
import sys
input = sys.stdin.readline

def solve():
    H, W = map(int, input().split())
    blocks = list(map(int, input().split()))
    
    total_water = 0
    
    # 맨 앞과 맨 뒤는 물이 고일 수 없으므로 1부터 W-2까지 탐색
    for i in range(1, W - 1):
        # 기준점 i의 왼쪽과 오른쪽에서 각각 가장 높은 블록을 찾음
        left_max = max(blocks[:i])
        right_max = max(blocks[i+1:])
        
        # 물이 차오르는 경계선은 둘 중 낮은 높이
        boundary = min(left_max, right_max)
        
        # 현재 바닥 높이가 경계선보다 낮다면 그 차이만큼 빗물 추가
        if boundary > blocks[i]:
            total_water += boundary - blocks[i]
            
    print(total_water)

if __name__ == "__main__":
    solve()

```

### 🔍 요약 (체크 포인트)

1. **가로(`W`) 기준으로 세로 기둥 하나씩 쪼개어 보기:** 2차원으로 복잡하게 생각할 필요 없이, "지금 내가 서 있는 칸 위로 하늘에서 물이 떨어지면 어느 높이까지 차오를까?"만 고민하면 됩니다.
2. **`H` 변수의 사용 유무:** 문제 입력으로 전체 높이 $H$가 주어지지만, 이 풀이 방식에서는 각 칸의 상대적 높이만 중요하므로 사실상 $H$ 변수를 쓰지 않고 풀 수 있습니다.

이 문제는 투 포인터(Two Pointers) 알고리즘을 사용하면 시간 복잡도 $O(W)$로 더 빠르게 최적화할 수도 있습니다. 혹시 이 투 포인터 최적화 방법에 대해서도 추가로 알아보고 싶으신가요?
