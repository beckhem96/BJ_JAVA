백준 15685번 **드래곤 커브** 문제는 복잡해 보이는 기하학적 형태를 **"방향의 규칙성"**을 찾아 단순한 수열로 바꾸는 것이 핵심인 아주 훌륭한 시뮬레이션 문제입니다.

이 문제에서 커브를 90도씩 회전시켜서 끝점에다 이어 붙인다는 설명을 그대로 구현(좌표를 회전시키는 등)하려고 하면 수식이 매우 복잡해집니다. 대신 **"어느 방향으로 이동했는지"** 그 순서만 기록하면 마법처럼 쉽게 풀립니다!

### 💡 핵심 아이디어: 이동 방향의 규칙 (뒤집고 + 1)

문제의 방향 조건은 다음과 같습니다.

* `0`: 오른쪽 ($\rightarrow$)
* `1`: 위쪽 ($\uparrow$)
* `2`: 왼쪽 ($\leftarrow$)
* `3`: 아래쪽 ($\downarrow$)

시작 방향이 `0`인 드래곤 커브의 세대별 이동 방향을 적어보겠습니다.

* **0세대:** `[0]`
* **1세대:** `[0, 1]`
* **2세대:** `[0, 1, 2, 1]`
* **3세대:** `[0, 1, 2, 1, 2, 3, 2, 1]`

규칙이 보이시나요? 다음 세대를 만들 때는 **이전 세대의 방향 기록을 뒤에서부터 거꾸로 읽으면서 1씩 더해주면(+90도 회전)** 됩니다. (만약 값이 4가 되면 0으로 순환합니다.)

1. **이전 세대:** `[0, 1]`
2. **뒤에서부터 읽기:** `1` 그 다음 `0`
3. **1씩 더하기 (+90도):** `2` 그 다음 `1`
4. **원래 기록에 이어 붙이기:** `[0, 1]` + `[2, 1]` = `[0, 1, 2, 1]`

이렇게 각 드래곤 커브의 모든 이동 방향을 구한 뒤, 지도(101x101 배열)에 선을 그어가며 꼭짓점을 방문 처리(`true`) 해주기만 하면 됩니다.

### 1. Java 풀이

격자의 크기가 $100 \times 100$이므로 점의 좌표는 $0$부터 $100$까지 가능합니다. 따라서 배열의 크기를 `101`로 잡아야 합니다.

```java
import java.io.*;
import java.util.*;

public class Main {
    // 방향: 0(우), 1(상), 2(좌), 3(하)
    static int[] dx = {1, 0, -1, 0};
    static int[] dy = {0, -1, 0, 1};
    static boolean[][] map = new boolean[101][101];

    public static void main(String[] args) throws IOException {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        int N = Integer.parseInt(br.readLine());

        for (int i = 0; i < N; i++) {
            StringTokenizer st = new StringTokenizer(br.readLine());
            int x = Integer.parseInt(st.nextToken());
            int y = Integer.parseInt(st.nextToken());
            int d = Integer.parseInt(st.nextToken()); // 시작 방향
            int g = Integer.parseInt(st.nextToken()); // 세대

            drawDragonCurve(x, y, d, g);
        }

        // 1x1 정사각형의 네 꼭짓점이 모두 드래곤 커브의 일부인지 확인
        int count = 0;
        for (int i = 0; i < 100; i++) {
            for (int j = 0; j < 100; j++) {
                if (map[i][j] && map[i + 1][j] && map[i][j + 1] && map[i + 1][j + 1]) {
                    count++;
                }
            }
        }

        System.out.println(count);
    }

    static void drawDragonCurve(int x, int y, int d, int g) {
        List<Integer> dirs = new ArrayList<>();
        dirs.add(d); // 0세대 방향 추가

        // 1. 드래곤 커브의 세대만큼 방향 수열 만들기
        for (int i = 0; i < g; i++) {
            int size = dirs.size();
            // 뒤에서부터 거꾸로 읽으면서 +1 (90도 시계방향 회전)
            for (int j = size - 1; j >= 0; j--) {
                dirs.add((dirs.get(j) + 1) % 4);
            }
        }

        // 2. 완성된 방향 수열을 바탕으로 지도에 점 찍기
        map[x][y] = true; // 시작점 마킹
        int cx = x;
        int cy = y;
        
        for (int dir : dirs) {
            cx += dx[dir];
            cy += dy[dir];
            map[cx][cy] = true; // 이동한 꼭짓점 마킹
        }
    }
}

```

### 2. Python 풀이

파이썬에서는 리스트 슬라이싱(`[::-1]`)을 활용하면 뒤집기 연산을 매우 직관적이고 우아하게 처리할 수 있습니다.

```python
import sys

input = sys.stdin.readline

def solve():
    N = int(input())
    
    # 101 x 101 격자 (좌표가 0~100)
    board = [[False] * 101 for _ in range(101)]
    
    # 방향: 우, 상, 좌, 하
    dx = [1, 0, -1, 0]
    dy = [0, -1, 0, 1]
    
    for _ in range(N):
        x, y, d, g = map(int, input().split())
        
        # 0세대 방향
        dirs = [d]
        
        # 1. 세대만큼 방향 수열 생성 (뒤집어서 1씩 더하기)
        for _ in range(g):
            # dirs[::-1]로 뒤집은 후 각 요소에 +1 하고 4로 나눈 나머지
            dirs += [(dir + 1) % 4 for dir in dirs[::-1]]
            
        # 2. 격자에 드래곤 커브 그리기 (방문 처리)
        board[x][y] = True
        cx, cy = x, y
        for dir in dirs:
            cx += dx[dir]
            cy += dy[dir]
            board[cx][cy] = True

    # 3. 네 꼭짓점이 모두 True인 1x1 정사각형 개수 세기
    count = 0
    for i in range(100):
        for j in range(100):
            if board[i][j] and board[i+1][j] and board[i][j+1] and board[i+1][j+1]:
                count += 1
                
    print(count)

if __name__ == "__main__":
    solve()

```

### 🔍 요약 (체크 포인트)

1. **좌표계 주의:** 일반적인 2차원 배열 문제는 행(row)이 $y$, 열(col)이 $x$지만, 이 문제는 수학 좌표계처럼 가로가 $x$, 세로가 $y$로 주어집니다. 다행히 $100 \times 100$의 정방형 격자이므로 배열을 `map[x][y]` 형태로 직관적으로 매핑해도 전혀 문제가 없습니다.
2. **배열 크기:** 범위가 $0 \le x, y \le 100$이므로 배열은 반드시 크기를 **101**로 잡아주어야 `IndexOutOfBounds` 에러를 피할 수 있습니다.
3. **방향의 규칙(뒤집어서 +1):** 기하학적인 회전을 단순히 "방향 수열의 규칙성"으로 바꿔서 푼다는 발상이 이 문제의 99%를 차지합니다.

리스트를 뒤집어서 방향을 추가하는 방식이 어떻게 90도 회전을 완벽하게 모사하는지 구조적으로 더 궁금하신 부분이 있다면 언제든 편하게 말씀해주세요!

가장 헷갈리는 이유는 **"리스트(`dirs`)에 값을 계속 추가하고 있는데, 도대체 어디서부터 어디까지 읽어서 추가하는 건지"** 때문일 거예요.

이 코드가 정확히 어떻게 돌아가는지, 시작 방향이 `0`이고 2세대(`g=2`)까지 그리는 상황을 가정해서 **한 줄 한 줄 시뮬레이션**을 해보겠습니다.

### 🎬 변수 추적 시뮬레이션 (시작: `dirs = [0]`)

**▶ `i = 0` (1세대를 만드는 중)**

1. `int size = dirs.size();`
* 현재 `dirs` 안에는 `[0]` 하나만 있으므로 **`size = 1`**입니다.
* *이 코드가 핵심입니다! 앞으로 값이 추가되더라도, "이번 세대에서는 딱 이 인덱스까지만 읽겠다"고 기준선을 그어두는 역할입니다.*


2. `for (int j = 0; j >= 0; j--)`
* `size - 1`인 `0`번 인덱스부터 거꾸로 읽기 시작합니다.
* `j = 0`일 때: `dirs.get(0)`은 `0`입니다. 여기에 1을 더하면 1.
* `dirs.add(1 % 4)` $\to$ `dirs`에 **1**을 추가합니다.


3. **1세대 완료 결과:** `dirs = [0, 1]`

**▶ `i = 1` (2세대를 만드는 중)**

1. `int size = dirs.size();`
* 현재 `dirs` 안에는 `[0, 1]` 두 개가 있으므로 **`size = 2`**입니다.


2. `for (int j = 1; j >= 0; j--)`
* `size - 1`인 `1`번 인덱스부터 거꾸로(`1 -> 0` 순서로) 읽기 시작합니다.
* `j = 1`일 때: `dirs.get(1)`은 `1`입니다. 1을 더하면 2.
  $\to$ `dirs.add(2 % 4)` $\to$ `dirs`에 **2**를 추가합니다. (현재 `dirs = [0, 1, 2]`)
* `j = 0`일 때: `dirs.get(0)`은 `0`입니다. 1을 더하면 1.
  $\to$ `dirs.add(1 % 4)` $\to$ `dirs`에 **1**을 추가합니다.


3. **2세대 완료 결과:** `dirs = [0, 1, 2, 1]`

### 💡 핵심 포인트 두 가지

1. **`int size = dirs.size();` 를 안에서 매번 구하지 않고 밖으로 빼둔 이유:**
   만약 `for(int j = dirs.size()-1; ...)` 처럼 반복문 조건식에 바로 넣어버리면, 내가 값을 하나 추가할 때마다 리스트의 길이가 늘어나서 **반복문이 끝나지 않는 무한 루프**에 빠지거나 엉뚱한 값을 읽게 됩니다. 그래서 값을 추가하기 전에 "이번에 읽을 분량"을 `size`라는 변수에 박아두는 것입니다.
2. **`% 4` 를 하는 이유:**
   방향은 0(우), 1(상), 2(좌), 3(하) 밖에 없습니다. 만약 현재 방향이 `3(하)`일 때 시계방향으로 90도 꺾으면 다시 `0(우)`이 되어야 합니다.
   `3 + 1 = 4`인데, 방향 번호에는 4가 없죠? 그래서 `4 % 4 = 0`으로 만들어서 **방향 번호가 0, 1, 2, 3 안에서 계속 빙글빙글 돌도록** 만들어주는 안전장치입니다.
