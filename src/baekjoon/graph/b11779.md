백준 11779번 **최소비용 구하기 2** 문제는 우리가 이전에 풀었던 두 가지 문제의 핵심 개념이 합쳐진 아주 훌륭한 종합 문제입니다.

1. **다익스트라(Dijkstra):** '녹색 옷 입은 애가 젤다지?' 문제처럼 최소 비용(최단 거리)을 구해야 합니다.
2. **경로 역추적(Path Reconstruction):** '숨바꼭질 4' 문제처럼 내가 어떤 경로를 거쳐서 왔는지 부모 노드를 기록해야 합니다.

### 💡 핵심 아이디어: 다익스트라 + 경로 기록 배열

최소 비용을 구하는 과정은 일반적인 다익스트라 알고리즘과 완전히 똑같습니다. 단지, **비용이 갱신될 때마다 "어디서부터 왔는지"를 기록**하는 배열(`route` 또는 `prev`)을 하나 더 추가해주면 됩니다.

1. **다익스트라 수행:** 우선순위 큐를 이용해 시작 도시부터 각 도시까지의 최소 비용 배열(`dist`)을 갱신합니다.
2. **경로 기록 (`route[next] = current`):** 만약 `next` 도시로 가는 더 싼 경로를 찾아서 `dist[next]`를 갱신했다면, 그 직전 도시가 `current`라는 것을 `route` 배열에 저장합니다.
3. **경로 역추적:** 도착 도시(`end`)부터 시작해서 `route` 배열을 타고 거슬러 올라갑니다. 시작 도시에 도달할 때까지 이 과정을 반복하며 스택이나 리스트에 담은 뒤, 역순으로 출력하면 정답이 됩니다.

### 1. Java 풀이

```java
import java.io.*;
import java.util.*;

public class Main {
    // 도착 도시와 비용을 저장할 클래스
    static class Node implements Comparable<Node> {
        int to, cost;

        public Node(int to, int cost) {
            this.to = to;
            this.cost = cost;
        }

        @Override
        public int compareTo(Node o) {
            return Integer.compare(this.cost, o.cost); // 비용 기준 오름차순 정렬
        }
    }

    public static void main(String[] args) throws IOException {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        
        int n = Integer.parseInt(br.readLine()); // 도시의 개수
        int m = Integer.parseInt(br.readLine()); // 버스의 개수

        List<List<Node>> graph = new ArrayList<>();
        for (int i = 0; i <= n; i++) {
            graph.add(new ArrayList<>());
        }

        // 간선 정보 입력
        for (int i = 0; i < m; i++) {
            StringTokenizer st = new StringTokenizer(br.readLine());
            int u = Integer.parseInt(st.nextToken());
            int v = Integer.parseInt(st.nextToken());
            int w = Integer.parseInt(st.nextToken());
            graph.get(u).add(new Node(v, w));
        }

        // 출발점, 도착점 입력
        StringTokenizer st = new StringTokenizer(br.readLine());
        int start = Integer.parseInt(st.nextToken());
        int end = Integer.parseInt(st.nextToken());

        // 다익스트라 초기 세팅
        int[] dist = new int[n + 1];
        int[] route = new int[n + 1]; // 경로 역추적 배열
        Arrays.fill(dist, Integer.MAX_VALUE);

        PriorityQueue<Node> pq = new PriorityQueue<>();
        pq.offer(new Node(start, 0));
        dist[start] = 0;

        // 다익스트라 알고리즘 시작
        while (!pq.isEmpty()) {
            Node current = pq.poll();

            // 도착 지점에 도달하면 탐색 종료 (최소 비용 보장)
            if (current.to == end) break;

            // 이미 처리된 적 있는 노드라면 무시 (가지치기)
            if (current.cost > dist[current.to]) continue;

            for (Node next : graph.get(current.to)) {
                int nextCost = current.cost + next.cost;

                // 더 적은 비용으로 갈 수 있는 길을 찾았다면 갱신
                if (nextCost < dist[next.to]) {
                    dist[next.to] = nextCost;
                    route[next.to] = current.to; // ★ 핵심: 어디서 왔는지 기록
                    pq.offer(new Node(next.to, nextCost));
                }
            }
        }

        // 1. 최소 비용 출력
        System.out.println(dist[end]);

        // 2. 경로 역추적 (도착지 -> 출발지)
        Stack<Integer> stack = new Stack<>();
        int curr = end;
        while (curr != 0) { // 출발지의 부모는 0이므로 0이 될 때까지 반복
            stack.push(curr);
            curr = route[curr];
        }

        // 3. 경로에 포함된 도시 개수 출력
        System.out.println(stack.size());

        // 4. 경로 출력 (스택에서 빼면서 역순으로)
        StringBuilder sb = new StringBuilder();
        while (!stack.isEmpty()) {
            sb.append(stack.pop()).append(" ");
        }
        System.out.println(sb.toString().trim());
    }
}

```

### 2. Python 풀이

```python
import sys
import heapq

input = sys.stdin.readline
INF = int(1e9)

def solve():
    n = int(input())
    m = int(input())
    
    graph = [[] for _ in range(n + 1)]
    
    for _ in range(m):
        u, v, w = map(int, input().split())
        graph[u].append((v, w))
        
    start, end = map(int, input().split())
    
    dist = [INF] * (n + 1)
    route = [0] * (n + 1) # 경로를 기록할 배열
    
    pq = [(0, start)] # (비용, 현재 도시)
    dist[start] = 0
    
    while pq:
        current_cost, current_node = heapq.heappop(pq)
        
        # 도착 지점 도달 시 조기 종료
        if current_node == end:
            break
            
        # 기존에 있는 거리보다 현재 뽑아낸 거리가 더 길다면 무시
        if current_cost > dist[current_node]:
            continue
            
        for next_node, weight in graph[current_node]:
            next_cost = current_cost + weight
            
            # 더 짧은 경로를 발견하면 갱신
            if next_cost < dist[next_node]:
                dist[next_node] = next_cost
                route[next_node] = current_node # ★ 이전 노드 기록
                heapq.heappush(pq, (next_cost, next_node))
                
    # 1. 최소 비용 출력
    print(dist[end])
    
    # 2. 경로 역추적
    path = []
    curr = end
    while curr != 0:
        path.append(curr)
        curr = route[curr]
        
    # 3. 도시의 개수 출력
    print(len(path))
    
    # 4. 경로 출력 (리스트를 뒤집어서 출력)
    print(*(path[::-1]))

if __name__ == "__main__":
    solve()

```

### 🔍 풀이 시 주의할 점 (실수 포인트)

1. **출발지의 `route` 처리:** 배열을 0으로 초기화해 두면, 시작점의 `route[start]` 값은 0이 됩니다. 도시 번호는 1번부터 시작하므로 역추적할 때 `curr != 0`을 조건으로 삼으면 시작점까지 완벽하게 스택(또는 리스트)에 담을 수 있습니다.
2. **조기 종료 조건:** 우선순위 큐를 사용했으므로 큐에서 도착점(`end`)을 꺼내는 순간, 그 경로가 최단 경로임이 보장됩니다. 이때 `break`로 탈출해주면 탐색 시간을 상당히 단축시킬 수 있습니다.
3. **중복 간선:** A에서 B로 가는 버스가 여러 대일 수 있습니다. 하지만 인접 리스트에 그대로 다 넣어도 다익스트라 로직상 최솟값만 알아서 갱신되므로 특별히 예외 처리를 할 필요는 없습니다.

다익스트라와 경로 추적을 결합하는 흐름은 완벽히 이해가 되셨나요? 혹시 역추적 과정에서 `route` 배열의 값이 어떻게 변해가는지 구체적인 예시로 확인해보고 싶으신 부분이 있다면 편하게 말씀해주세요!

### 1. `if (current.cost > dist[current.to]) continue;` 의 진짜 의미

말씀하신 대로 **"내가 지금 큐에서 꺼낸 경로의 비용이, 이미 알고 있는 최소 비용보다 비싸면 그냥 버린다"**는 뜻이 맞습니다.

그런데 여기서 한 가지 의문이 들 수 있습니다.
*"애초에 최솟값만 `dist`에 넣고 큐에 넣었을 텐데, 왜 큐에서 꺼낸 값이 `dist`보다 큰 상황이 발생할까?"*

그 이유는 우선순위 큐(Min-Heap)의 특성 때문입니다. 큐 안에는 **'과거에 기록된 비싼 티켓'**이 아직 남아있기 때문이죠.

**[상황 예시]**

1. 서울 $\to$ 대전 경로를 탐색하다가 비용이 **50**인 경로를 발견했습니다.
* `dist[대전] = 50`으로 갱신하고, 큐에 `(대전, 50)`을 넣습니다.


2. 다른 길로 서울 $\to$ 대전 경로를 탐색하다가 비용이 **30**인 더 싼 경로를 발견했습니다!
* `dist[대전] = 30`으로 갱신하고, 큐에 `(대전, 30)`을 넣습니다.



이제 큐 안에는 대전으로 가는 티켓이 두 장 `(대전, 30)`, `(대전, 50)` 들어있습니다.
우선순위 큐는 싼 것부터 꺼내므로 `(대전, 30)`을 먼저 꺼내서 다음 도시들을 탐색합니다.

한참 뒤에 큐에서 아까 남겨진 `(대전, 50)`이 나옵니다.
이때 `current.cost`는 **50**이고, 현재 기록된 `dist[대전]`은 **30**입니다.
이미 30원에 갈 수 있는 걸 아는데 굳이 50원짜리 경로를 이어서 탐색할 필요가 없겠죠?
그래서 이 `if`문을 만나면 **"어? 이거 옛날에 넣었던 비싼 티켓이네? 버려!"** 하고 `continue`로 무시해 버리는 것입니다. (이를 '가지치기'라고 합니다.)

### 2. 중복 간선: "버스가 여러 대여도 예외 처리가 필요 없는 이유"

이제 두 번째 질문인 **"A에서 B로 가는 중복 간선을 왜 그냥 다 리스트에 때려 넣어도 괜찮은가?"**에 대한 답이 나옵니다.

바로 **1번에서 설명한 '우선순위 큐'와 '가지치기(`if`문)'가 알아서 비싼 버스를 걸러주기 때문**입니다.

**[상황 예시]**
입력으로 서울 $\to$ 부산 버스가 3대 주어졌다고 해봅시다.

* 버스 1: 비용 100
* 버스 2: 비용 40
* 버스 3: 비용 70

이 3대를 인접 리스트에 그냥 다 넣습니다. 그리고 서울에서 출발해 이웃 도시를 탐색할 때, 이 3대의 버스를 모두 확인합니다.

1. 버스 1을 봅니다: `nextCost = 100`. (현재 `dist[부산]`은 무한대) $\to$ `dist[부산]`을 **100**으로 갱신하고 큐에 넣음.
2. 버스 2를 봅니다: `nextCost = 40`. (현재 `dist[부산]`은 100) $\to$ 더 싸니까 `dist[부산]`을 **40**으로 갱신하고 큐에 넣음.
3. 버스 3을 봅니다: `nextCost = 70`. (현재 `dist[부산]`은 40) $\to$ **더 비싸니까 아예 갱신도 안 하고 큐에 넣지도 않음.**

자, 이 탐색이 끝나면 큐에는 `(부산, 40)`과 `(부산, 100)` 두 개가 들어있게 됩니다.

* 우선순위 큐이므로 당연히 `(부산, 40)`이 먼저 나와서 탐색을 이어갑니다.
* 나중에 `(부산, 100)`이 큐에서 나오면? 아까 1번에서 배운 `if (current.cost > dist[current.to])` 조건문에 딱 걸려서 **아무것도 안 하고 바로 버려집니다.**

### 💡 최종 요약

A에서 B로 가는 노선이 수백 개라도, 다익스트라 알고리즘은 **가장 싼 노선 정보만 큐의 앞쪽으로 밀어 올려서 먼저 처리**합니다. 그리고 나중에 뒤늦게 나오는 비싼 중복 노선들은 `continue` 조건문이 전부 쓰레기통에 버려줍니다.

그래서 중복 간선을 걸러내려고 입력받을 때 굳이 끙끙댈 필요 없이, 리스트에 편하게 다 담아도 알고리즘이 완벽하게 최소 비용만 찾아내는 것입니다!
