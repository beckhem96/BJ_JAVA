백준 4179번 **불!** 문제는 BFS를 두 번(또는 두 개의 큐로 동시에) 돌려야 하는 아주 흥미로운 **다중 BFS(Multi-source BFS)** 문제입니다.

지훈이도 이동하지만 불도 매분마다 사방으로 퍼지기 때문에, **"불이 먼저 번진 곳이나 동시에 번지는 곳은 지훈이가 갈 수 없다"**는 조건이 핵심입니다.

### 💡 핵심 아이디어: 불의 시간 vs 지훈이의 시간

가장 안전하고 확실한 방법은 **불에 대한 BFS를 먼저 다 돌려놓고, 그 정보를 바탕으로 지훈이의 BFS를 돌리는 것**입니다.

1. **불의 BFS (`fire_time`):**
* 초기 불의 위치(들)를 모두 큐에 넣습니다. (불이 여러 개일 수 있다는 점에 주의!)
* BFS를 돌면서 각 칸에 불이 도달하는 최소 시간을 기록합니다.


2. **지훈이의 BFS (`jihoon_time`):**
* 초기 지훈이의 위치를 큐에 넣습니다.
* 다음 칸으로 이동할 때, 지훈이가 도달할 시간(`지훈이의 현재 시간 + 1`)이 **불이 도달할 시간보다 엄격하게 작을 때만** 이동할 수 있습니다.


3. **탈출 조건:**
* 지훈이가 미로의 가장자리(행이 0이거나 $R-1$, 열이 0이거나 $C-1$)에 도달한 상태에서 한 번 더 이동하면 탈출 성공입니다!

### 1. Java 풀이

```java
import java.io.*;
import java.util.*;

public class Main {
    static int R, C;
    static char[][] maze;
    static int[][] fireTime;
    static int[][] jihoonTime;
    static int[] dx = {-1, 1, 0, 0};
    static int[] dy = {0, 0, -1, 1};

    public static void main(String[] args) throws IOException {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        StringTokenizer st = new StringTokenizer(br.readLine());

        R = Integer.parseInt(st.nextToken());
        C = Integer.parseInt(st.nextToken());

        maze = new char[R][C];
        fireTime = new int[R][C];
        jihoonTime = new int[R][C];

        Queue<int[]> fireQ = new LinkedList<>();
        Queue<int[]> jihoonQ = new LinkedList<>();

        // 배열 초기화
        for (int i = 0; i < R; i++) {
            Arrays.fill(fireTime[i], Integer.MAX_VALUE); // 불이 안 닿는 곳은 무한대로
            Arrays.fill(jihoonTime[i], -1); // 지훈이 방문 여부는 -1로
        }

        for (int i = 0; i < R; i++) {
            String line = br.readLine();
            for (int j = 0; j < C; j++) {
                maze[i][j] = line.charAt(j);
                if (maze[i][j] == 'F') {
                    fireQ.offer(new int[]{i, j});
                    fireTime[i][j] = 0;
                } else if (maze[i][j] == 'J') {
                    jihoonQ.offer(new int[]{i, j});
                    jihoonTime[i][j] = 0;
                }
            }
        }

        // 1. 불에 대한 BFS
        while (!fireQ.isEmpty()) {
            int[] cur = fireQ.poll();
            int cx = cur[0];
            int cy = cur[1];

            for (int dir = 0; dir < 4; dir++) {
                int nx = cx + dx[dir];
                int ny = cy + dy[dir];

                if (nx < 0 || ny < 0 || nx >= R || ny >= C) continue;
                if (maze[nx][ny] == '#' || fireTime[nx][ny] != Integer.MAX_VALUE) continue;

                fireTime[nx][ny] = fireTime[cx][cy] + 1;
                fireQ.offer(new int[]{nx, ny});
            }
        }

        // 2. 지훈이에 대한 BFS
        while (!jihoonQ.isEmpty()) {
            int[] cur = jihoonQ.poll();
            int cx = cur[0];
            int cy = cur[1];

            // 가장자리에 도달했다면 다음 번 이동으로 탈출 가능
            if (cx == 0 || cy == 0 || cx == R - 1 || cy == C - 1) {
                System.out.println(jihoonTime[cx][cy] + 1);
                return;
            }

            for (int dir = 0; dir < 4; dir++) {
                int nx = cx + dx[dir];
                int ny = cy + dy[dir];

                if (nx < 0 || ny < 0 || nx >= R || ny >= C) continue;
                if (maze[nx][ny] == '#' || jihoonTime[nx][ny] >= 0) continue;

                // 핵심: 지훈이가 도착할 시간이 불이 도착할 시간보다 작아야만 이동 가능!
                if (jihoonTime[cx][cy] + 1 < fireTime[nx][ny]) {
                    jihoonTime[nx][ny] = jihoonTime[cx][cy] + 1;
                    jihoonQ.offer(new int[]{nx, ny});
                }
            }
        }

        // 큐가 다 빌 때까지 탈출을 못 했다면
        System.out.println("IMPOSSIBLE");
    }
}

```

### 2. Python 풀이

```python
import sys
from collections import deque

input = sys.stdin.readline

def solve():
    R, C = map(int, input().split())
    
    maze = []
    fire_q = deque()
    jihoon_q = deque()
    
    fire_time = [[float('inf')] * C for _ in range(R)]
    jihoon_time = [[-1] * C for _ in range(R)]
    
    for i in range(R):
        row = list(input().strip())
        maze.append(row)
        for j in range(C):
            if row[j] == 'F':
                fire_q.append((i, j))
                fire_time[i][j] = 0
            elif row[j] == 'J':
                jihoon_q.append((i, j))
                jihoon_time[i][j] = 0

    dx = [-1, 1, 0, 0]
    dy = [0, 0, -1, 1]

    # 1. 불 BFS
    while fire_q:
        cx, cy = fire_q.popleft()
        
        for i in range(4):
            nx = cx + dx[i]
            ny = cy + dy[i]
            
            if 0 <= nx < R and 0 <= ny < C:
                if maze[nx][ny] != '#' and fire_time[nx][ny] == float('inf'):
                    fire_time[nx][ny] = fire_time[cx][cy] + 1
                    fire_q.append((nx, ny))

    # 2. 지훈 BFS
    while jihoon_q:
        cx, cy = jihoon_q.popleft()
        
        # 가장자리면 다음 번에 바로 탈출
        if cx == 0 or cy == 0 or cx == R - 1 or cy == C - 1:
            print(jihoon_time[cx][cy] + 1)
            return

        for i in range(4):
            nx = cx + dx[i]
            ny = cy + dy[i]
            
            if 0 <= nx < R and 0 <= ny < C:
                if maze[nx][ny] != '#' and jihoon_time[nx][ny] == -1:
                    # 지훈이가 불보다 먼저 도착할 수 있는 경우만 이동
                    if jihoon_time[cx][cy] + 1 < fire_time[nx][ny]:
                        jihoon_time[nx][ny] = jihoon_time[cx][cy] + 1
                        jihoon_q.append((nx, ny))

    print("IMPOSSIBLE")

if __name__ == "__main__":
    solve()

```

### 🚨 많이 하는 실수 (오답 노트)

1. **불이 아예 없는 경우:** 입력 미로에 불(`F`)이 단 하나도 없을 수 있습니다. 이 경우 불의 도달 시간을 모두 `무한대(Integer.MAX_VALUE 또는 float('inf'))`로 초기화해두어야, 지훈이가 이동 조건을 통과할 수 있습니다. (0으로 초기화하면 지훈이가 이동을 못 합니다.)
2. **불이 여러 개인 경우:** 처음부터 큐에 불의 위치를 모두 넣고 BFS를 시작해야 합니다. 한 불이 다 퍼지고 나서 다른 불을 퍼뜨리면 시간이 꼬이게 됩니다.
3. **지훈이가 처음부터 가장자리에 있는 경우:** 지훈이를 큐에서 꺼내자마자 가장자리인지 확인하고 정답을 출력하는 구조여야, 1분 만에 탈출하는 경우를 예외 없이 잡을 수 있습니다.

불과 지훈이의 방문 시간을 독립적인 2차원 배열로 관리하는 방식이 깔끔한데, 혹시 두 배열을 비교하는 부분에서 헷갈리는 점이 있으신가요?

---
bfs이기 때문에 2차월 배열에 처음 입력된 값이 최단 시간이라고 판단한건지?

네, **정확하게 짚으셨습니다! 💯** BFS(너비 우선 탐색)를 사용했기 때문에, **어떤 칸에 처음 방문해서 `fireTime`을 기록하는 그 순간이 무조건 "최단 시간"임이 수학적으로 보장**됩니다.

그렇게 판단할 수 있는 이유를 BFS의 동작 원리와 함께 3가지로 나누어 아주 쉽게 설명해 드릴게요.

---

### 1. BFS는 "물결(동심원)"처럼 퍼져나간다

호수에 돌을 던지면 물결이 중심에서부터 둥글게 퍼져나가죠? BFS도 똑같습니다.

* **0초:** 처음에 불이 있는 위치들을 모두 확인합니다. (거리 0)
* **1초:** 0초인 위치들에서 딱 **한 칸** 떨어진 곳들을 모두 확인합니다. (거리 1)
* **2초:** 1초인 위치들에서 딱 **한 칸** 떨어진 곳들을 모두 확인합니다. (거리 2)

이처럼 BFS는 **거리가 가까운 곳(시간이 적게 걸리는 곳)부터 완벽하게 다 탐색한 뒤에야, 그다음 거리로 넘어갑니다.**

### 2. 큐(Queue)의 FIFO(선입선출) 구조

BFS는 큐를 사용하기 때문에 **"먼저 큐에 들어간 놈이 먼저 나온다"**는 절대 규칙을 따릅니다.

만약 큐에 `[1초에 도달한 칸들]`이 들어있다면, 이 칸들을 다 꺼내서 처리하기 전까지는 `[2초에 도달한 칸들]`이 절대 먼저 처리될 수 없습니다.
시간이 역전되는 일이 큐 자료구조 안에서는 일어날 수 없는 것이죠.

### 3. "처음 만났다 = 그 전에는 올 수 없었다"

위의 두 가지 특징을 합쳐보면 명확한 결론이 나옵니다.

어떤 특정 칸 `(r, c)`에 불이 **3초** 만에 처음 도착했다고 가정해 봅시다.
이 말은 즉,

* 0초일 때의 불도 이 칸에 못 닿았고,
* 1초일 때의 불도 이 칸에 못 닿았고,
* 2초일 때의 불도 이 칸에 못 닿았다는 뜻입니다.

따라서 불이 이 칸에 도달할 수 있는 **가장 빠른 시간(최단 시간)은 무조건 3초**가 되는 것입니다. 나중에 다른 불길이 빙글빙글 돌아서 이 칸에 5초 만에 도착하더라도, 이미 3초라는 최단 시간이 기록되어 있으므로 무시(`continue`)하면 됩니다.
