백준 1647번 **도시 분할 계획** 문제는 **최소 스패닝 트리(MST, Minimum Spanning Tree)** 알고리즘을 응용하는 아주 좋은 문제입니다.

이 문제의 핵심 목표는 "마을을 두 개로 분할하되, 유지비의 합을 최소로 만드는 것"입니다.

### 💡 핵심 아이디어: "MST를 구하고, 가장 비싼 간선 하나를 끊어라!"

1. **마을을 하나로 연결하기 (MST):**
* 먼저 모든 집들을 최소한의 비용으로 모두 연결합니다. 이때 사용되는 알고리즘이 바로 **크루스칼(Kruskal)** 또는 **프림(Prim)** 알고리즘입니다.
* 이렇게 하면 $N$개의 집이 $N-1$개의 길로 연결된 **최소 스패닝 트리**가 완성됩니다.


2. **마을을 두 개로 분할하기:**
* 트리의 특성상, 트리 안에서 **간선(길)을 하나만 끊으면 무조건 두 개의 분리된 트리(마을)로 나뉘게 됩니다.**
* 비용을 최소로 만들어야 하므로, 우리가 만든 MST에서 **가장 유지비가 비싼 길 하나를 없애면** 정답이 됩니다!



크루스칼 알고리즘은 가중치 기준으로 오름차순 정렬을 한 뒤 간선을 선택하므로, **MST에 마지막으로 추가되는 간선이 무조건 가장 비용이 큰 간선**이 됩니다.

### 1. Java 풀이 (크루스칼 알고리즘)

유니온 파인드(Union-Find)를 활용하여 크루스칼 알고리즘을 구현합니다.

```java
import java.io.*;
import java.util.*;

public class Main {
    // 간선 정보를 담을 클래스
    static class Edge implements Comparable<Edge> {
        int a, b, cost;

        public Edge(int a, int b, int cost) {
            this.a = a;
            this.b = b;
            this.cost = cost;
        }

        @Override
        public int compareTo(Edge o) {
            return Integer.compare(this.cost, o.cost); // 비용 기준 오름차순 정렬
        }
    }

    static int[] parent;

    public static void main(String[] args) throws IOException {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        StringTokenizer st = new StringTokenizer(br.readLine());

        int N = Integer.parseInt(st.nextToken()); // 집의 개수
        int M = Integer.parseInt(st.nextToken()); // 길의 개수

        List<Edge> edges = new ArrayList<>();
        for (int i = 0; i < M; i++) {
            st = new StringTokenizer(br.readLine());
            int a = Integer.parseInt(st.nextToken());
            int b = Integer.parseInt(st.nextToken());
            int cost = Integer.parseInt(st.nextToken());
            edges.add(new Edge(a, b, cost));
        }

        // 간선을 비용 순으로 오름차순 정렬
        Collections.sort(edges);

        // 부모 테이블 초기화 (자기 자신을 부모로)
        parent = new int[N + 1];
        for (int i = 1; i <= N; i++) {
            parent[i] = i;
        }

        int totalCost = 0;
        int maxEdge = 0; // MST에 포함된 간선 중 가장 비용이 큰 간선

        // 크루스칼 알고리즘 수행
        for (Edge edge : edges) {
            // 사이클이 발생하지 않는 경우에만 집합에 포함
            if (find(edge.a) != find(edge.b)) {
                union(edge.a, edge.b);
                totalCost += edge.cost;
                maxEdge = edge.cost; // 오름차순 정렬이므로 마지막으로 들어간 값이 최댓값
            }
        }

        // 전체 MST 비용에서 가장 비싼 간선의 비용을 뺌 (마을 2개로 분할)
        System.out.println(totalCost - maxEdge);
    }

    // 부모 노드를 찾는 함수 (경로 압축)
    static int find(int x) {
        if (parent[x] == x) return x;
        return parent[x] = find(parent[x]);
    }

    // 두 집합을 합치는 함수
    static void union(int a, int b) {
        int rootA = find(a);
        int rootB = find(b);
        if (rootA != rootB) {
            parent[rootB] = rootA;
        }
    }
}

```

### 2. Python 풀이 (크루스칼 알고리즘)

파이썬에서도 튜플을 리스트에 담아 `sort()` 함수를 사용하면 직관적으로 구현할 수 있습니다.

```python
import sys
input = sys.stdin.readline

# 부모 노드 찾기 (경로 압축)
def find(parent, x):
    if parent[x] != x:
        parent[x] = find(parent, parent[x])
    return parent[x]

# 두 집합 합치기
def union(parent, a, b):
    root_a = find(parent, a)
    root_b = find(parent, b)
    if root_a != root_b:
        parent[root_b] = root_a

def solve():
    N, M = map(int, input().split())
    
    edges = []
    for _ in range(M):
        a, b, cost = map(int, input().split())
        edges.append((cost, a, b)) # 정렬을 위해 cost를 제일 앞에 둠
        
    # 비용을 기준으로 오름차순 정렬
    edges.sort()
    
    # 부모 테이블 초기화
    parent = [i for i in range(N + 1)]
    
    total_cost = 0
    max_edge = 0
    
    # 크루스칼 알고리즘 진행
    for cost, a, b in edges:
        # 사이클이 발생하지 않을 때만 간선 선택
        if find(parent, a) != find(parent, b):
            union(parent, a, b)
            total_cost += cost
            max_edge = cost # 정렬되어 있으므로 계속 덮어씌우면 마지막이 최댓값
            
    # 전체 MST 비용에서 가장 비싼 간선 하나를 제거하여 두 마을로 분할
    print(total_cost - max_edge)

if __name__ == "__main__":
    solve()

```

### 🔍 요약 (핵심 포인트)

* **어떤 알고리즘을 쓸까?** 간선의 가중치 합을 최소화하며 모든 노드를 연결해야 하므로 **MST 알고리즘(크루스칼)**을 사용합니다.
* **마을 분할의 트릭:** $N$개의 집을 2개의 묶음으로 나누려면, 정확히 **$N-2$개의 간선**만 선택하면 됩니다. 크루스칼 알고리즘은 오름차순으로 간선을 선택하므로, 평소처럼 MST를 만들고 **제일 마지막에 연결된(가장 비싼) 간선 하나만 취소**하면 깔끔하게 가장 저렴한 두 개의 마을이 탄생합니다.

질문하신 부분은 이 문제에서 **가장 많은 분들이 헷갈려 하는 문장**입니다! 아주 예리한 질문이에요.

결론부터 말씀드리면, **경로를 여러 개 지우는 것이 맞습니다.** 하지만 우리가 코드로 구현한 **'크루스칼 알고리즘'이 알아서 그 불필요한 길들을 다 지워주고(무시하고) 있었던 것**입니다.

이해하기 쉽게 쪼개서 설명해 드릴게요.

그리고 각 분리된 마을 안에서도 임의의 두 집 사이에 경로가 항상 존재하게 하면서 길을 더 없앨 수 있다.  이거 때문에 경로를 여러개 지우는 줄 알았는데 아니야?
### 1. "길을 더 없앨 수 있다"의 진짜 의미

문제의 조건들을 하나씩 번역해 보면 이렇습니다.

* "임의의 두 집 사이에 경로가 항상 존재하게 한다" $\to$ **모든 집이 연결되어 있어야 한다.**
* "유지비의 합을 최소로 하면서 길을 없앤다" $\to$ **불필요한 지름길이나 빙 돌아가는 길(사이클)을 모두 없애서 최소한의 길만 남긴다.**

이 두 가지를 합치면 정확히 **'최소 스패닝 트리(MST)'의 정의**가 됩니다. 즉, 마을 안에 불필요한 도로가 없도록 **최소한의 뼈대**만 남기라는 뜻입니다.

### 2. 크루스칼 알고리즘은 이미 길을 버리고 있다

입력으로 집이 N개, 길이 M개 주어지죠? (보통 M이 N보다 훨씬 큽니다.)

우리가 `if (find(a) != find(b))` 조건으로 간선을 선택할 때, 크루스칼 알고리즘은 딱 **N - 1개의 길만 선택**하고 나머지 수많은 길들은 쳐다보지도 않고 버립니다.

이 과정 자체가 이미 문제에서 말한 **"각 분리된 마을 안에서도 (...) 길을 더 없앨 수 있다"**를 수행하고 있는 것입니다! (사이클을 만드는 불필요한 길들을 큐에서 꺼내기만 하고 채택하지 않았으니까요.)

### 3. 왜 마지막에 간선 '하나'만 뺄까?

자, 크루스칼 알고리즘이 끝난 직후의 상태를 상상해 보세요.
수많은 도로(M개)가 다 지워지고, 마을 전체가 딱 **N - 1개의 도로로 아슬아슬하게 모두 연결된 거대한 트리(Tree) 1개**가 되었습니다.

여기서 트리의 아주 중요한 수학적 성질이 나옵니다.

> **"트리에서 간선(길)을 단 1개만 끊으면, 무조건 두 개의 완벽한 트리로 분리된다."**

마을을 2개로 쪼개야 하니까, 이미 군더더기가 다 제거된 거대한 트리에서 **길을 딱 하나만 더 자르면** 자연스럽게 두 개의 마을로 나뉘게 됩니다.
그리고 유지비를 최소로 해야 하니까, 우리가 고른 N - 1개의 길 중에서 **가장 비싼 길 1개**를 자르는 것입니다.
